# Матрица соответствия руководству Inobitek DICOM Viewer 2.18 и приоритеты AIVision

Документ фиксирует статус функциональности относительно структуры пользовательского руководства (*Viewer_ru.pdf*, оглавление) и **не** означает обязательство полного клонирования продукта Inobitek. Приоритеты привязаны к [клиническим требованиям](./clinical-requirements.md).

## Явно вне скоупа продукта

| Область | Примечание |
|--------|------------|
| **Гл. 1 — инфраструктура PACS-просмотрщика** | Меню «Сеть / Хранилище», поиск с пресетами, панели загрузки с сервера, CD/DVD/zip как источник, экспорт списка исследований, многооконность десктоп-класса. **AIVision: веб + локальная папка** — это осознанное ограничение, в матрице не оценивается. |

## Легенда статусов

| Статус | Значение |
|--------|----------|
| **Есть** | Покрывает типовой сценарий главы на уровне, достаточном для ежедневной работы в нашем продукте. |
| **Частично** | Есть базовая реализация или задел; поведение или охват существенно уже, чем в мануале. |
| **Нет** | Не реализовано. |

## Приоритизация по [clinical-requirements.md](./clinical-requirements.md)

Порядок из раздела «Приоритет внедрения» (п. 7) трактуется как **скелет дорожной карты**. Ниже — как к нему «подвешены» блоки из мануала Inobitek **без** копирования всех пунктов глав 2–10 в одну фазу.

### P0 — Критично для заявленного клинического ядра

Соответствует: viewer одной серии, пресеты режимов, **динамика двух исследований**, задел под MPR/3D.

| Направление | Связь с мануалом (гл.) | Задачи уровня AIVision |
|-------------|-------------------------|-------------------------|
| Динамика | 2 (синхронизация, мультиокна) | Довести сравнение до требований п. 3 *clinical-requirements*: синхронный zoom/pan (если ещё не полностью), даты, общий пресет; затем — следующая версия (привязка по области, заметки). |
| Режимы просмотра | 2 (W/L), 4.1–4.2 клиндок | Расширить и стабилизировать пресеты (лёгкие, бронхи, сосуды, кость, …) и их связь с MPR/3D где заявлено. |
| Базовый 2D/MPR | 2, 5 | Стабильная ортогональная связка курсора/индексов, предсказуемая геометрия (уже частично в коде). |
| Похожие случаи | вне мануала | Минимальный сценарий из п. 2 *clinical-requirements*. |

### P1 — Углубление viewer и подготовка к «workstation-light»

Соответствует: MPR и 3D по дорожной карте, плюс инструменты, которые напрямую поддерживают режимы «сосуды / опухоль / кость» и отчётность.

| Направление | Связь с мануалом | Заметки |
|-------------|------------------|---------|
| Измерения и аннотации | 2, 5 | Начать с набора, нужного клиндоку: линейные + углы + полигон/площадь + HU/ROI-статистика на срезе; затем расширение. |
| Интерполяция, отражение, теги DICOM | 2 | Низкая сложность / высокая отдача для доверия к viewer. |
| Cine (2D) | 2 | Для динамических серий и контроля режимов. |
| 3D: crop / видимость | 3 | Куб видимости или эквивалент (обрезка объёма) — ключ к «3D как в задаче», без полного набора кистей. |
| Ориентация | 2.30.1, 3 | LPS-куб или пресеты стандартных видов (согласовать с уже имеющимися осями в VTK). |
| Экспорт снимка / серии | 2.28 | Растр для отчёта; DICOM secondary capture — по мере необходимости регуляторики. |

### P2 — Fusion, продвинутая МПР, сегментация RT-уровня

Соответствует: п. 6–7 *clinical-requirements* (сегментация и аналитика).

| Направление | Связь с мануалом |
|-------------|------------------|
| Объединение серий, регистрация, fusion | 4 |
| CPR, толщина среза, маркеры MPR | 5 |
| Панель структур, маски, boolean, RT I/O | 6 |
| Запись видео, продвинутые фильтры, DSA, ПЭТ-модуль, сосуды/сердце | 2, 3, 7–10 |

### P3 / отложено до отдельного решения

- Полный **анализ сосудов** (гл. 7), **сердце** (8), **ПЭТ** (9), **эндоскопия** (10+).
- **Перфузия**, **кровоток МРТ**, **сшивка**, **SUV**, **кальций КА**, **ADC**, **УЗИ-измерения** — включать точечно только если появится модальность/регуляторное требование в продукте.

## Как пользоваться документом

1. **Статусы** обновлять при завершении эпиков (глобально по главе и по строкам детализации).
2. **Приоритеты** пересматривать при изменении `clinical-requirements.md`, а не при каждом новом подпункте мануала Inobitek.
3. Гл. 1 **намеренно не ведётся** — это зафиксированный скоуп продукта.
