# Дорожная карта

## Фаза 1. MVP просмотра КТ

Цель:
- дойти до первого реально работающего просмотрщика DICOM.

Шаги:
1. Создать каркас приложения на React + TypeScript + Vite.
2. Подключить загрузку локальных DICOM-файлов.
3. Научиться выделять исследования и серии.
4. Показать первый срез.
5. Реализовать стек срезов.
6. Добавить zoom, pan, window/level.
7. Добавить линейку и измерения.

## Фаза 2. Диагностический режим

Цель:
- сделать интерфейс ближе к реальной работе радиолога.

Шаги:
1. Панель серий.
2. Панель метаданных исследования.
3. Горячие клавиши.
4. Пресеты окна: кость, легкие, мягкие ткани.
5. Синхронизация элементов управления.
6. Расширенные клинические режимы: сосуды, бронхи, сердце, суставы, опухоли.
7. Подготовка интерфейса к сравнению в динамике.

## Фаза 3. MPR и 3D

Цель:
- получить объемную работу с КТ.

Шаги:
1. Построить volume из серии.
2. Реализовать аксиальный, сагиттальный и корональный окна.
3. Добавить 3D rendering на vtk.js.
4. Добавить базовые пресеты визуализации.
5. Добавить профильные 3D-пресеты для костей, суставов и сосудов.

## Фаза 4. Сравнение в динамике

Цель:
- дать врачу возможность сравнивать текущее и предыдущее исследование.

Шаги:
1. Загрузка двух исследований одного пациента.
2. Side-by-side viewer.
3. Синхронный scroll.
4. Синхронный zoom/pan.
5. Синхронизация window/level и режимов просмотра.

## Фаза 5. Похожие случаи

Цель:
- добавить систему поддержки врача через похожие случаи и вероятные паттерны.

Шаги:
1. Отдельная панель похожих случаев.
2. Выбор области интереса или текущего среза.
3. Поиск похожих паттернов.
4. Показ уровня уверенности и краткого объяснения.
5. Открытие похожего случая рядом с текущим.

## Фаза 6. Сегментация

Цель:
- перейти от просмотра к построению полезных моделей.

Шаги:
1. Threshold mask.
2. ROI.
3. Редактирование маски.
4. Подготовка к экспорту 3D-модели.
