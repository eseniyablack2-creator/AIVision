# Radiology Workstation Benchmark

Дата: 2026-04-04

## Что берем как эталон

### RadiAnt
- Очень быстрый базовый 2D workflow без перегруженного UI.
- Понятные CT-пресеты окна.
- Сильная связка `2D + MPR + 3D VR`.
- Практичный `PET-CT fusion` и измерение `SUV`.
- Обрезка объема в `3D VR` без сложного лишнего интерфейса.

### OsiriX MD
- Полноценный workstation-подход, а не просто viewer.
- `Hanging Protocols`, многомониторность, 4D, PET-CT.
- `Curved MPR` для аорты, бронхов и извитых структур.
- 3D post-processing как отдельный клинический слой, а не декоративный режим.
- Локальная база, PACS, экспорт, key images и ROI как часть рабочего контура.

### Horos
- Open-source и хороший ориентир по расширяемой архитектуре.
- Подход “ядро viewer + плагины + сообщество”.
- Важен как эталон открытой, модульной медицинской платформы.

### Visage 7
- Сильнейший enterprise workflow.
- Синхронизация и linking между окнами и исследованиями.
- Регистрация и linked scrolling между MPR-группами.
- Подход `One Viewer`, где радиолог не переключается между разными приложениями.

## Что внедряем в AIVision

### 1. Базовый диагностический слой
- Максимум полезной площади под изображение.
- Минимум панелей сверху.
- Основные инструменты и пресеты в компактном правом toolbar.
- Быстрое переключение `1x1 / 2x2 / MPR`.
- Предсказуемая синхронизация crosshair между окнами.

### 2. MPR слой
- Корректные `axial / coronal / sagittal`.
- Затем `oblique MPR`.
- Затем `curved MPR` для сосудов, аорты, бронхов и позвоночника.

### 3. 3D слой
- Настоящий `volume rendering`.
- Клинические transfer functions: `bone`, `soft tissue`, `vascular`, `lung`, `airways`.
- `Hide bones`, `auto table removal`, `clipping plane`.
- Управление вращением, зумом и обрезкой без потери рабочего пространства.

### 4. Сосудистый слой
- CTA-представление ближе к ангиографическому.
- Bone subtraction без грубой потери сосудов.
- Диаметр, плотность, профиль по сосуду, центральная линия.
- Подсветка подозрительных зон стеноза, кальция, аневризм.

### 5. PET-CT слой
- Fusion CT + PET.
- Поддержка `SUVmin / SUVmax / SUVmean`.
- ROI и подсветка патологического накопления.

### 6. Prior comparison
- Side-by-side.
- Linked scrolling.
- Регистрация.
- Разностное изображение и динамика изменений.

## Что не копируем
- Перегруженные верхние тулбары.
- Дублирование одних и тех же кнопок в нескольких местах.
- UI, который занимает больше внимания, чем сами изображения.

## Архитектурный вывод

AIVision должен идти не по пути “все кнопки сразу”, а по пути:

1. Надежный 2D viewer.
2. Надежный MPR.
3. Настоящий 3D VR.
4. Клинические специализированные пакеты.
5. Сравнение в динамике.
6. AI-подсказки и количественный анализ.
