# Техническая модель 3D-визуализации сосудов в AIVision и в ПО класса Inobitek / VIDAR

Документ описывает, **как устроен текущий прототип** в этом репозитории, и **как обычно устроены** «режимы анализа сосудов» в коммерческих DICOM-станциях (обобщённо, без привязки к закрытому коду конкретного вендора).

## 1. Что делает AIVision сегодня

### 1.1 Данные

- Серия DICOM декодируется в браузере; по метаданным (положение срезов, ориентация, размер пикселя) строится **регулярная 3D-сетка** (воксельный объём) в `buildCtVolumeFromSeries` (`frontend/src/lib/ctVolume.ts`).
- Значения вокселей — **HU** в `Float32Array`, при необходимости с **даунсэмплингом** и **обрезкой стола** (эвристики по краям серии).

### 1.2 Рендеринг (vtk.js)

- Используется **vtk.js**: `vtkImageData` + `vtkVolumeMapper` + `vtkVolume` (`frontend/src/components/VolumeViewport.tsx`).
- Режим **CTA 3D** (`workspaceMode === 'cta3d'`):
  - **DVR** (composite): классический прямой объёмный рендеринг — вдоль луча накапливается цвет и непрозрачность по **передаточным функциям** (opacity + color transfer function).
  - **MIP**: по лучу берётся **максимальная** интенсивность (хорошо для яркого контраста в сосудах).
  - **AVG**: усреднение вдоль луча (грубый аналог «толстого slab» без отдельной геометрии слоя).
- Режим **Airways 3D**: `MinimumIntensityProjection` и TF, заточенные под низкие HU (воздушные структуры).

Передача HU → цвет/прозрачность задаётся в `configureVolumeRendering` (`frontend/src/lib/vtkCtTransferFunctions.ts`): кусочно-линейные функции по шкале HU, с учётом:

- ползунков **«Сосуды»** / **«Кость−»** и кнопки **«Кости»** (подавление плотной кости в TF — это **не** анатомическая сегментация, а **масштабирование вклада** диапазонов HU);
- **сдвига HU** и **яркости объёма** (множитель непрозрачности), добавленных для управления DVR в 3D-режиме отдельно от 2D W/L.

### 1.3 Взаимодействие

- Камера vtk по умолчанию: **вращение ЛКМ**, **масштаб колесом**, **панорамирование средней кнопкой**.
- На контейнер вешается `wheel` с `preventDefault` (**passive: false**), чтобы колесо **не уводило скролл страницы** и доходило до vtk.
- **Зонд**: луч из экранных координат в мир, затем трассировка по сетке HU (`frontend/src/lib/volumeRayProbe.ts`); Shift+отпускание ЛКМ синхронизирует перекрестье MPR.

### 1.4 Чего нет (как у Inobitek Vessel Analysis / VIDAR «Сосуды туловища»)

Типичный коммерческий модуль добавляет цепочку **после** или **вместе** с VR/MIP:

1. **Сегментация сосудов** (часто обучаемая модель или классический vesselness + пороги).
2. **Центральная линия (centerline)** — граф вдоль оси сосуда.
3. **Straightened / stretched CPR** — «развёртка» сосуда вдоль centerline в 2D.
4. **Поперечные сечения** и графики **диаметра / площади** вдоль длины.
5. **Удаление кости** в КТА: от **двухэтапного вычитания** (натив + контраст) до **ABS** (automatic bone suppression) на ИИ или морфологии.

В AIVision эти этапы **не реализованы**; есть только объёмная визуализация и эвристическое приглушение кости в TF.

## 2. Как это обычно делают «по-взрослому» (обзор)

### 2.1 Построение объёма

Как и здесь: DICOM → упорядоченный 3D-массив, часто с **трилинейной интерполяцией** при нерегулярном шаге срезов. На GPU данные кладут в **3D-текстуру** для ray casting.

### 2.2 Удаление кости (bone removal / subtraction)

- **Двойное сканирование + вычитание**: маска кости с нативной серии вычитается из контрастной; чувствительно к движению пациента.
- **Морфология / пороги по HU**: быстро, но ломается у черепа, ребер, металла.
- **Обучаемая сегментация кости** (3D U-Net и аналоги): современный вариант ABS для КТА головы/шеи и др.
- **Multiscale методы**: уменьшают «съедание» сосудов у границы с костью.

### 2.3 Выделение сосудов

- **Фильтры на основе Гессиана** (Frangi / подобные) — многошкальный vesselness.
- **Порог по HU** после контраста + морфология — грубо, но быстро.
- **Сегментационные сети** (TotalSegmentator / специализированные CTA-модели).

### 2.4 Рендеринг

- **Ray casting** по 3D-текстуре с **transfer function** (как в vtk DVR).
- **MIP / MinIP / AVG** как частные случаи редукции вдоль луча.
- Иногда **изосurface** по порогу для «оболочки» сосуда после сегментации.

### 2.5 Centerline и CPR

После бинарной маски сосуда строят **скелет** / **минимальное дерево** / **fast marching** по расстоянию до стенки, затем **сглаживание** кривой. Вдоль centerline вырезают **стек поперечных срезов** (перпендикулярно касательной) и склеивают в **straightened image**.

## 3. Связь с сервером TotalSegmentator

Сегментация органов и структур (в т.ч. сосудистых классов в зависимости от задачи) может выполняться **на Python** (`inference/`) через TotalSegmentator; результат — **multilabel NIfTI** (`POST /v1/upload-nifti` или путь к файлу на сервере). **Наложение** этой маски на vtk-объём в браузере в текущем UI **ещё не подключено** — это следующий этап интеграции.
